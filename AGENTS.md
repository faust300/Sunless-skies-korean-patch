# Sunless Skies 한국어 패치 작업 지침

이 저장소에서는 번역 작업의 속도와 흐름을 우선한다. 작은 변경마다 전체 검사, 빌드, 설치 시험, 커밋, 푸시를 반복하지 않는다.

## 작업 문서 사용

- 긴 대화 기록에 의존하지 말고 `docs/tasks/README.md`에서 현재 작업에 해당하는 문서만 읽는다.
- 모든 작업 문서를 매번 읽지 않는다. 번역이면 번역 문서, 배포면 배포 문서만 확인한다.
- 작업 중 새로 확정된 규칙, 용어, 경로 또는 주의점은 해당 작업 문서에 짧게 갱신한다.
- 일회성 진행 상황은 문서에 쌓지 않는다. 다음 작업에서도 필요한 결정과 재현 절차만 기록한다.
- 작업 종류가 둘 이상이면 필요한 문서만 조합해서 사용한다.

| 작업 | 우선해서 읽을 문서 |
| --- | --- |
| 일반 번역 | `docs/tasks/translation.md` |
| 스크린샷 제보 처리 | `docs/tasks/screenshot-review.md` |
| 용어 통일 | `docs/tasks/terminology.md` |
| 설치기·직접 패치 | `docs/tasks/installer.md` |
| 품질 검사 | `docs/tasks/qa.md` |
| 커밋·푸시·Release | `docs/tasks/release.md` |

## 기본 작업 방식

- 번역 수정은 로컬 작업 트리에 계속 누적한다.
- 사용자가 스크린샷을 제시하면 화면에 보이는 원문뿐 아니라 같은 이벤트, 선택지, 설명 및 인접 문자열도 함께 검색한다.
- 용어집과 기존 번역을 먼저 확인하고 이미 채택한 표기를 유지한다.
- 원문의 변수, 숫자 자리표시자, `[dir:...]` 같은 동적 토큰, 색상 및 강조 태그는 보존한다.
- 번역 파일은 UTF-8 인코딩을 유지한다.
- 사용자가 요청하지 않은 번역 문장이나 기존 변경을 되돌리지 않는다.

## 검증 수준

### 일반 번역 및 문서 수정

- 수정한 항목의 형식과 주변 문맥만 확인한다.
- 전체 번역 사전 검사, 전체 빌드, 설치기 실행, 게임 파일 검사까지 수행하지 않는다.
- 단순 문서 수정은 내용 확인만 하고 별도의 빌드 검증을 하지 않는다.

### 구조에 영향을 주는 변경

다음 변경에만 관련 도구나 좁은 범위의 검사를 수행한다.

- 설치기 또는 패처 코드 변경
- 번역 파일 파서 형식 변경
- 정적 에셋이나 글꼴 교체
- 배포 패키지 구조 변경
- 변수, 태그 또는 동적 토큰을 대량 수정하는 작업

검사는 변경된 범위에 맞게 최소화하고, 문제가 발견될 때만 범위를 넓힌다.

### 배포 전 검증

사용자가 명시적으로 배포를 요청한 경우에만 다음 절차를 한 번 수행한다.

1. 누적 변경 확인
2. 번역 파일 형식과 주요 토큰 검사
3. 설치기 빌드 및 패키지 구성 확인
4. 버전과 변경 이력 갱신
5. 커밋 및 푸시
6. 태그 생성과 GitHub Release 확인

## Git 및 배포 규칙

- 사용자가 `푸시`, `커밋`, `릴리스`, `배포`를 명시하기 전에는 해당 작업을 수행하지 않는다.
- 여러 번역 수정은 하나의 작업 묶음으로 취합한다.
- 번역 한두 건마다 버전을 올리거나 Release를 만들지 않는다.
- 사용자가 `푸시하자`라고 하면 누적 변경을 검토해 한 번에 커밋하고 푸시한다.
- 사용자가 `릴리스하자`라고 하면 버전 확인 후 태그와 Release까지 진행한다.
- 이미 공개된 태그를 임의로 이동하거나 덮어쓰지 않는다.

## 사용자 응답

- 중간 진행 설명은 짧게 유지한다.
- 단순 수정에는 긴 계획이나 반복 상태 보고를 하지 않는다.
- 완료 응답에는 변경한 내용과 수정 파일만 간단히 알린다.
- 커밋, 푸시, 릴리스를 하지 않았다면 했다고 표현하지 않는다.
- 시간이 오래 걸리는 검사나 빌드가 필요할 때만 그 이유를 먼저 알린다.

## 번역 우선순위

1. 사용자가 스크린샷으로 제보한 미번역 및 오역
2. 제보된 문자열과 연결된 이벤트 및 선택지
3. 메뉴, 도움말, 상태 표시 등 반복 노출되는 시스템 UI
4. 용어 불일치와 명백한 기계 번역
5. 장문의 문학적 표현과 세부 문체 검수
