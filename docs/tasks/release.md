# 커밋 및 배포

## 실행 조건

- `커밋`, `푸시`, `배포`, `릴리스`를 사용자가 명시한 경우에만 수행한다.
- 일반 번역 중에는 변경을 로컬에 누적한다.

## 푸시 절차

1. 누적된 변경 파일을 확인한다.
2. 변경 범위에 맞는 최소 검사를 수행한다.
3. README의 번역 수치와 변경 이력이 필요한지 확인한다.
4. 관련 파일만 스테이징해 하나의 의미 있는 커밋으로 만든다.
5. `main`을 푸시하고 작업 트리 상태를 확인한다.

## Release 절차

1. 새 버전 번호를 정한다.
2. README와 설치기 프로젝트 버전을 동일하게 갱신한다.
3. 배포 전 QA를 한 번 수행한다.
4. 커밋과 푸시를 완료한다.
5. 새 태그를 만들고 푸시한다.
6. GitHub Actions의 `Release Installer` 성공 여부를 확인한다.
7. Release에 `SunlessSkies-KoreanPatch.zip`이 등록됐는지 확인한다.

## 규칙

- 이미 공개된 태그를 이동하거나 강제 덮어쓰지 않는다.
- 번역 몇 건만으로 새 Release를 만들지 않는다.
- 변경 이력에는 실제로 수행한 번역, 구조 변경 및 검증만 기록한다.
- Release 설명은 README의 해당 버전 변경 이력과 일치시킨다.
