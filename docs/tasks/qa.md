# 품질 검사

## 기본 원칙

일반 번역 중에는 전체 검사를 실행하지 않는다. 사용자가 검증을 요청하거나 배포 직전일 때만 필요한 검사를 모아서 한 번 수행한다.

## 번역 검사

- UTF-8 깨짐과 비정상 문자
- 미번역 영문 문장
- 동일 원문의 상충 번역
- 변수와 숫자 자리표시자 손실
- `[dir:...]` 등 동적 토큰 손실
- 색상 및 강조 태그 불일치
- 용어집과 다른 고유명사

## 패키지 검사

- 설치기 빌드 성공 여부
- `payload` 번역 파일과 정적 에셋 포함 여부
- ZIP 내부 실행 파일 및 폴더 구조
- 실제 게임 경로 탐색과 백업 동작
- 이미 적용된 파일에 대한 중복 치환 여부

## 기록

- 큰 검수 결과만 날짜가 포함된 `docs/translation-qa-YYYYMMDD.md`에 남긴다.
- 단순 오탈자 수정마다 QA 보고서를 만들지 않는다.
- 실패나 남은 위험을 성공으로 숨기지 않고 짧게 기록한다.
