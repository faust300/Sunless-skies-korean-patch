# 스크린샷 기반 번역 작업 규칙

스크린샷에서 미번역 문구가 발견되면 해당 문구 하나만 번역하지 않는다. 문자열의 실제 저장 위치를 확인하고 같은 화면, 에셋, 데이터 레코드에 속한 인접 문자열을 함께 추출하고 번역한다.

## 조사 순서

1. 스크린샷의 영문 문자열을 원문 그대로 검색한다.
2. 저장 위치를 `global-metadata.dat`, Unity 에셋, 스토리 `.bytes`, LocalLow 캐시 중 하나로 분류한다.
3. 발견한 오브젝트나 레코드의 앞뒤 문자열과 동일 UI 구성 요소의 형제 항목을 함께 추출한다.
4. 화면에 노출될 가능성이 있는 미번역 문자열을 한 묶음으로 번역한다.
5. 동적 문장은 고정 문구와 숫자 앞뒤 조각, 날짜 서식, 복수형을 함께 확인한다.
6. 패치 후 해당 묶음에 남은 영문 UI 문자열을 다시 보고서로 출력한다.

## 묶음 예시

- `Fuel` 또는 `Supplies`: 제목, 보유 용기 수, 현재 용기 잔량, 단수·복수형
- 날짜: 긴 날짜와 짧은 날짜 서식, 월 이름, 서수 접미사
- 조언 창: 제목, 본문, `조언 끄기`, `조언 숨기기`와 같은 모든 버튼
- 능력치 창: 레벨, 경험치, 선택 가능한 특성 수, 안내 문구
- 항구 화면: 장소명, `출항`, `탐험`, 툴팁과 지역명
- 메인 메뉴: 도움말 제목, 본문, 링크 문구, 버전 표기를 제외한 주변 UI

## 검수 원칙

- 화면에 보이는 문구만 보고 저장 위치를 추측하지 않는다.
- 이미 번역된 항목도 용어와 문장 연결이 자연스러운지 함께 확인한다.
- 오브젝트명이나 런타임 조회 키는 사용자 표시 문자열로 확인되지 않는 한 변경하지 않는다.
- 한글이 길어지는 화면은 우선 번역을 적용하고 실제 화면에서 줄바꿈과 크기를 별도로 조정한다.
