# 지도 접근 문구 스크린샷 검수 (2026-07-12)

## 제보 화면

막달렌 병원 인근을 이동할 때 다음 영어 문구가 화면 중앙에 표시되었다.

```text
Magdalene's - a place of peace, contemplation and rest. The Driver twitches miserably.
```

이 문구는 일반 지도 지명이나 지역 설명이 아니라, 특정 항구 또는 지역에 접근했을 때 현재 장교가 반응하는 지역 접근 대사다.

## 번역 키 위치

현재 번역은 다음 파일에 존재한다.

- 파일: `translations/eueeeeeeeek_legacy_missing_20260711.txt`
- 위치: 617행

```text
Magdalene's - a place of peace, contemplation and rest. The Driver twitches miserably.=막달렌 병원. 평화와 명상, 휴식의 장소입니다. 조종사가 비참하게 움찔거립니다.
```

기존 한글 패치 비교 자료에도 같은 영어 원문 키가 존재한다.

- `_korean_patch/compare_existing_ko/BepInEx/Translation/ko/Text/translation.txt` 622행
- `_korean_patch/compare_existing_ko/BepInEx/plugins/SunlessKoTranslator/translation.txt` 623행

## 현재 패치 적용 구조

v1.1.1부터 번역 사전은 저장소 최상위 `translations` 폴더에 보관한다. 현재 패치는 BepInEx 런타임 번역 방식이 아니라 설치기가 번역 사전을 읽어 게임 데이터 파일을 직접 수정하는 방식이다.

- 번역 사전 입력: `translations/*.txt`
- 갱신 도구: `tools/Update-KoreanPatch.ps1`
- 직접 패치 데이터 출력: `payload/Sunless Skies_Data`
- Release 구성 시 `translations/*.txt`도 배포 패키지에 포함

따라서 번역 사전에 문장이 있다는 사실만으로 현재 설치된 게임 데이터에 반영되었다고 단정할 수 없다. 설치기 또는 갱신 도구가 이 키가 들어 있는 원본 데이터 파일을 찾아 치환해야 실제 게임 화면에 표시된다.

## 영어로 표시된 가능한 원인

1. 최신 번역 사전으로 게임을 다시 패치하지 않았다.
2. 직접 패치 도구가 해당 문구가 저장된 게임 데이터 파일을 패치 대상으로 찾지 못했다.
3. 게임 데이터의 실제 원문이 번역 키와 공백, 대시, 구두점 또는 태그 면에서 다르다.
4. 런타임에서 지역 설명과 장교 반응을 결합해 표시해, 저장된 원문 단위와 완성된 화면 문구가 다르다.
5. 이전에 패치된 게임 데이터가 영어 원문도 현재 한국어 키도 아닌 중간 형태로 남아 있다.

## 같은 묶음의 인접 대사

해당 파일의 610행대에는 막달렌 병원에 접근할 때 장교별로 출력되는 대사가 연속해서 들어 있다.

- 참회하는 악마: 막달렌 병원의 작은 위안에 관한 반응
- 공주: 목욕과 대화를 기대하며 정박을 요구하는 반응
- 신호수: 첨탑을 보고 기분이 밝아지는 반응
- 이모: 악몽을 치료하라며 정박을 권하는 반응
- 쥐 여단: 막달렌 병원의 색을 평하는 반응
- 기관사: 평화와 휴식이라는 설명에 괴로워하는 반응
- 방랑자: 이곳에 온 사람 중 운 좋은 이만 떠난다는 반응
- 벼려진 동반자: 영혼의 대장간과 비교하는 반응
- 기인: 진실이 쉽지는 않다고 평하는 반응

따라서 제보된 한 문장만 확인하지 말고 막달렌 병원 접근 대사 전체를 같은 조건에서 검사해야 한다. 다른 항구의 장교 접근 대사도 동일한 데이터 계열일 가능성이 높다.

## 번역 문장 검토

현재 번역:

```text
막달렌 병원. 평화와 명상, 휴식의 장소입니다. 조종사가 비참하게 움찔거립니다.
```

검토 사항:

- `Driver`는 기존 게임 용어에 맞춰 `기관사`로 옮기는 편이 적절하다.
- `twitches miserably`의 `비참하게 움찔거립니다`는 직역투가 강하다.
- 화면 중앙에 잠시 표시되는 문장이므로 짧고 즉시 이해되는 표현이 유리하다.

권장 번역:

```text
막달렌 병원. 평화와 명상, 휴식의 안식처입니다. 기관사는 괴로운 듯 몸을 떱니다.
```

더 짧은 대안:

```text
막달렌 병원. 평화와 명상, 휴식의 안식처입니다. 기관사는 괴롭게 몸을 떱니다.
```

## 인접 번역에서 발견된 추가 사항

인접 대사 일부에 조사 오류가 있다.

- `막달렌 병원를 바라봅니다`
- `막달렌 병원를 보고`

`병원를`은 `병원을`로 수정해야 한다. 같은 파일의 막달렌 병원 접근 대사를 일괄 적용하기 전에 장교 명칭, 조사와 문체도 함께 검수한다.

## 재현 및 확인 절차

1. 게임과 설치기를 종료한 상태에서 최신 번역 사전으로 직접 패치를 다시 생성한다.
2. 패치 보고서에서 제보된 영어 원문이 실제로 발견되고 치환되었는지 확인한다.
3. 동일한 세이브로 막달렌 병원에 접근해 화면 중앙 문구를 확인한다.
4. 장교를 바꾸어 막달렌 병원 접근 대사 여러 개가 한국어로 표시되는지 확인한다.
5. 여전히 영어라면 화면 문구를 그대로 캡처하고 실제 게임 데이터에서 정확한 런타임 문자열을 다시 추출한다.
6. 공백, 하이픈과 대시, 마침표, 따옴표 및 결합 전후 문자열 변형을 비교한다.

## 판정

- 번역 누락: 아님. 완성된 화면 문구와 일치하는 번역 키가 사전에 존재한다.
- 현재 게임 반영: 실패 또는 미확인. 제보 화면에서는 영어로 표시된다.
- 우선 확인 대상: 직접 패치 보고서의 원문 탐지 여부와 실제 게임 데이터 치환 여부
- 후속 번역 수정: `Driver` 용어, 직역투, 인접 문장의 조사 오류 정리

## 전수 조사 결과

- 도시·항구·랜드마크 접근 및 장교 반응 문구는 스토리 `.bytes`가 아니라 Unity `level*` 에셋에 저장되어 있다.
- 레거시 번역과 현재 게임 에셋을 대조한 결과 관련 문자열은 `level3`, `level4`, `level5`, `level6`, `level7`, `level9`, `level10`, `level11`, `level12`, `level13`, `level14`, `level16`, `level17`, `level20`, `level21`, `level22`, `level23`, `level24`에 분산되어 있다.
- 기존 배포 payload에는 이 가운데 일부 레벨만 포함되어 접근 문구 대부분이 영어로 남을 수 있었다.
- 정찰 문구는 튜토리얼 `level17`, 도움말 `resources.assets`, 발견 알림 `global-metadata.dat`으로 나뉜다.
